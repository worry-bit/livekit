# CloudPhone SDK H5 项目分析文档

## 1. 项目概述

**项目名称**: CloudPhoneClient  
**版本**: 1.2.36  
**技术栈**: 纯JavaScript + WebRTC + Socket.IO

这是一个云手机SDK的H5客户端，用于在浏览器中实现云手机功能。通过WebRTC技术实现远程视频流播放和输入控制，使用Socket.IO进行信令通信。

## 2. 项目结构

```

├── src/
│   ├── main.js              # SDK入口，导出createCloudPhonePlayer工厂函数
│   ├── CloudPlayer.js       # 核心播放器类（约2000+行）
```

## 3. 核心模块说明

### 3.1 CloudPlayer（核心类）

状态机定义：
```javascript
PlayerState = {
    NEW: 0,              // 新建
    RECONNECTING: 1,     // 重连中
    SIGNAL_CONNECTING: 2,// 信令连接中
    SIGNAL_CONNECTED: 3, // 信令已连接
    AUTHORIZED: 4,       // 已授权
    PEER_CONNECTING: 5,  // P2P连接中
    PEER_CONNECTED: 6,   // P2P已连接
    AUTOPLAY_FAILED: 7,  // 自动播放失败
    PLAYING: 8,          // 播放中
    PAUSED: 9,           // 暂停
    COUNTDOWN: 10,       // 倒计时中
    KEEPING: 11,         // 保持中
    CLOSED: 12           // 已关闭
}
```

### 3.2 Connection（信令连接）

负责与信令服务器通信：
- 使用 Socket.IO 建立 WebSocket 连接
- 处理授权认证
- 转发WebRTC信令（offer/answer/candidate）
- 管理房间会话

### 3.3 RTCSource（WebRTC管理）

负责WebRTC P2P连接：
- 创建和管理 RTCPeerConnection
- 处理 ICE 候选收集
- 统计信息（码率、帧率、延迟等）
- 音视频轨道管理

### 3.4 Input（输入控制）

支持多种输入类型：
- 鼠标事件（mousedown/mouseup/mousemove）
- 键盘事件（keydown/keyup）
- 触摸事件（touchstart/touchend/touchmove）
- 滚轮事件（wheel）
- 剪贴板同步

### 3.5 FileSender（文件传输）

通过DataChannel传输文件：
- 支持 File 对象和 Base64 字符串
- 分片传输（24KB/片）
- 进度回调

### 3.6 Sensors（传感器）

支持多种设备传感器：
- Accelerometer（加速度计）
- Gyroscope（陀螺仪）
- Gravity（重力）
- LinearAcceleration（线性加速度）
- Rotation（相对方向）
- GeomagneticRotation（地磁旋转）

### 3.7 Gps（定位）

使用 Geolocation API 获取位置信息并发送到云端。

### 4.2 WebRTC 建立链路（详细）

```
CloudPlayer._createPeer()
    │
    ├── 创建 RTCSource 实例
    │
    ├── 设置事件监听
    │   ├── connectionstatechange -> _onConnectionStateChange()
    │   ├── track -> _onTrack()
    │   ├── icecandidate -> _onLocalIceCandidate()
    │   ├── description -> _onLocalDescription()
    │   ├── datachannel -> _onDataChannel()
    │   └── statistics -> _onStatistics()
    │
    ├── source.open() 创建 RTCPeerConnection
    │
    ▼
connection.emitDataAsync("init", ...) 发送初始化参数
    │
    ▼
服务器返回 offer (通过信令通道)
    │
    ▼
_onRemoteMessage({type: "offer", payload})
    │
    ▼
RTCSource.setRemoteDescription(offer)
    │
    ├── 保存远端描述
    │
    ▼
createAnswer()
    │
    ├── _peerConn.createAnswer()
    ├── setLocalDescription()
    │
    ▼
emit("description", answer) 事件
    │
    ▼
_onLocalDescription() 发送 answer 到服务器
    │
    ▼
ICE 候选交换 (addRemoteIceCandidate / emit icecandidate)
    │
    ▼
PeerConnection 连接成功 ("connected")
    │
    ▼
_onPeerConnected()
    ├── 状态变为 PEER_CONNECTED
    ├── 启动统计
    ├── 创建文件发送器
    │
    ▼
视频轨道到达 (_onTrack)
    │
    ▼
video.srcObject = stream
    │
    ▼
_playVideo() 自动播放
    │
    ▼
状态变为 PLAYING
```

```

    ▼

    │

```

### 4.4 文件传输链路

```
用户调用 player.sendFile(file, dirPath, fileType, ...)
    │
    ▼
CloudPlayer._createFileSender()
    │
    ▼
FileSender.setFile() 设置文件
    │
    ▼
FileSender.open(peerConnection)
    │
    ▼
创建 DataChannel "file_upload"
    │
    ▼
FileSender.start()
    │
    ▼
_handleSendFileStart()
    │ 发送 WEB_FILE_START 消息（文件元数据）
    │
    ▼
_readNextChunk() 循环
    │
    ▼

    │ 分片发送（24KB/片）
    │ 发送 WEB_FILE_DATA 消息
    │
    ▼

    │ 发送 WEB_FILE_END 消息
    │
    ▼

```

### 4.5 重连机制链路

```
连接断开检测
    │
    ├── 信令断开 -> SignalState.DISCONNECTED
    └── P2P断开 -> connectionState = "disconnected" / "failed"
    │
    ▼
_onSignalConnectionStateChange() / _onPeerDisconnected()
    │
    ▼
检查条件
    │
    ├── 后台运行中 -> 延迟处理
    ├── 超过超时时间 -> 关闭
    └── 启用自动重连 -> _reconnect()
    │
    ▼
_reconnect(reason)
    │
    ├── 关闭摄像头/麦克风
    ├── 设置重连状态
    │
    ▼
根据 reason 执行
    │
    ├── "video_blocking" / "reconnect_by_user" / "ipv6_connect_failed"
    │   └── 重新开始流媒体
    │
    └── 其他 reason
        ├── 断开现有连接
        │
        ▼
        _connect() 重新连接信令服务器
```

## 6. 数据格式定义

### 6.1 输入事件格式

| 类型 | KeyType | 格式 |
|------|---------|------|
| 鼠标 | WEB_INPUT_MOUSE_ext | 长度(4) + 类型(1) + 时间戳(4) + 动作(2) + 按钮(1) + X(4) + Y(4) |
| 触摸 | WEB_INPUT_TOUCH_ext | 长度(4) + 类型(1) + 时间戳(4) + 动作(2) + 指头数(1) + [标识(4)+X(4)+Y(4)]*n |
| 键盘 | WEB_INPUT_KEY_ext | 长度(4) + 类型(1) + 时间戳(4) + 动作(2) + 键码(2) + 修饰符(2) |
| 滚轮 | WEB_INPUT_WHEEL_ext | 长度(4) + 类型(1) + 时间戳(4) + deltaY(4) + X(4) + Y(4) |
| GPS | WEB_INPUT_GPS | 长度(4) + 类型(1) + 时间戳(4) + 纬度(4) + 经度(4) + 海拔(4) + 精度(4) + 方向(4) + 速度(4) |
| 传感器 | WEB_INPUT_SENSOR | 长度(4) + 类型(1) + 时间戳(4) + 传感器类型(1) + 精度(1) + 值数量(1) + 值[n*4] |
| 文件开始 | WEB_FILE_START | JSON: {dirPath, fileName, fileType, size, extra} |
| 文件数据 | WEB_FILE_DATA | 二进制数据 |
| 文件结束 | WEB_FILE_END | JSON: {dirPath, fileName, fileType, size, extra} |

### 6.2 信令消息格式

| 类型 | 方向 | 说明 |
|------|------|------|
| init | H5→服务器 | 初始化参数（模式、角色、网络类型等） |
| offer/answer | 双向 | SDP 会话描述 |
| candidate | 双向 | ICE 候选 |
| start | H5→服务器 | 请求开始流媒体 |
| message | 双向 | 通用消息（streaming等） |
| close | H5→服务器 | 关闭会话 |

## 10. 依赖关系图

```
main.js (入口)
    │
```
